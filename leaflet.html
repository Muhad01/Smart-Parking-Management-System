Problem Context

In dense urban areas, drivers waste time and fuel searching for parking. Our Smart Parking Management System provides real‑time slot visibility, online bookings, and digital payments using a secure public‑cloud deployment.

CEA Objective

Architect and justify a secure, scalable, fault‑tolerant multi‑tier cloud solution (frontend, backend APIs, data store) without deploying live resources, but with an engineering‑grade design.

Cloud Architecture Overview

Provider: AWS (concepts map to Azure/GCP).

Multi‑Tier Design

  • Frontend: React/Next.js SPA on S3, served via CloudFront (HTTPS, CDN).
  • Backend APIs: Node.js/Express containers on ECS Fargate behind ALB.
  • Data Tier: Amazon RDS PostgreSQL (Multi‑AZ) + ElastiCache Redis.
  • Storage: S3 for logs, reports, backups.

Networks & Trust Boundaries

  • Dedicated VPC with 3 AZs.
  • Public subnets: ALB, NAT, CloudFront/WAF edge.
  • Private app subnets: ECS tasks (no direct Internet ingress).
  • Private data subnets: RDS & Redis, reachable only from app tier.
Data Flow
User → Browser → CloudFront/WAF → ALB → ECS APIs → RDS/Redis → response to user.

Reliability & Scalability

Reliability & Fault Tolerance

  • Multi‑AZ ECS tasks and RDS with automatic failover.
  • Health checks on ALB + ECS for self‑healing of failed tasks.
  • Automated backups, point‑in‑time restore, and audit logging.

SLOs: 99.9% uptime, p95 latency < 300 ms, error rate < 1%, RPO 5 min, RTO 30 min.

Scalability Plan

  • Assumed load: ~10k DAU, peak ≈ 100 RPS (3× safety margin).
  • ECS autoscaling: 3–10 tasks based on CPU and RPS per task.
  • RDS scaling: vertical scale + read replicas if read load grows.
  • Redis caching: parking availability & hot configuration to offload DB.
CLO1 & CLO2: Architecture design, reliability & scalability analysis with quantified SLOs and capacity planning.

Security & Compliance

Network & Access Control

  • Strict security groups; no database access from the Internet.
  • AWS WAF in front of CloudFront/ALB (OWASP Top 10, rate limiting).
  • Least‑privilege IAM roles for ECS, CI/CD, and DB access.

Encryption & Secrets

  • HTTPS everywhere (TLS 1.2+), HSTS, secure cookies.
  • RDS, ElastiCache, and S3 encrypted at rest via KMS.
  • All secrets (DB, JWT keys, API keys) in AWS Secrets Manager with rotation.

Operations, Monitoring & Cost

CI/CD & Monitoring

  • Git‑based flow with GitHub Actions / GitLab CI.
  • Pipeline: build → tests → security scan → staging → smoke tests → prod (blue‑green).
  • CloudWatch metrics, logs, dashboards, and alerts (errors, latency, CPU, DB health).
  • Runbooks for incidents: service outage, DB performance, payment gateway failure.

Cost & Optimization

  • Estimated baseline: ≈ $500–$850 / month (ECS, RDS, Redis, S3, CF, WAF, logs).
  • Optimization: rightsizing, Savings Plans/Reserved Instances, aggressive caching.
  • Cost controls: budgets + alerts, limit non‑prod runtime, cheaper storage tiers for logs.

Testing & Risk Management

Testing Strategy

  • Unit & integration tests for core APIs (bookings, payments, admin).
  • E2E browser tests for key user journeys.
  • Load tests (e.g., k6/JMeter) up to and beyond 100 RPS.
  • Chaos tests: task kill, RDS failover, simulated latency; verify SLOs.

Top Risks & Mitigations

  • DB bottleneck: indexes, Redis cache, vertical scale, read replicas.
  • Misconfig SG/IAM: IaC, reviews, principle of least privilege.
  • Cost spikes: budgets, dashboards, scheduled shutdowns for non‑prod.
  • DDoS / malicious traffic: WAF rules, rate limits, AWS Shield.
